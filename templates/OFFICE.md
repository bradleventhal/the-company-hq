The `id` and `timestamp` are auto-generated. A Loom-style screen recording is automatically captured when the server is running.

**What counts as an accomplishment:**
- Shipped a feature or fix
- Completed a research task or analysis
- Sent an approved outreach message
- Fixed a bug or resolved an issue
- Built an automation
- Wrote documentation or a guide
- Made a strategic decision with data

## тЪФя╕П Creating Quests (When You Need Human Input)

```bash

  -H "Content-Type: application/json" \
  -H "X-OpenClawfice-Token: $TOKEN" \

    "action": {
      "id": "<unique-id>",
      "type": "<type>",
      "icon": "<emoji>",
      "title": "<short title>",
      "description": "<what needs deciding>",
      "from": "<your name>",
      "priority": "<high|medium|low>",

  }'
```

**Quest types:**
- `approve_email` / `approve_send` тАФ Include `data.to`, `data.subject`, `data.body`
- `decision` with `data.options` тАФ e.g. `"options": ["Option A", "Option B"]`
- `decision` without options тАФ Free-form text input
- `input_needed` тАФ Text input with optional `data.placeholder`
- `review` / `review_data` тАФ Acknowledge button + optional notes

**Removing a completed quest:**
